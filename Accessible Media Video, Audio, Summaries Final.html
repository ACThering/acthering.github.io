Accessible Media: Video, Audio, Summaries

Use AI to accelerate routine text work, then complete manual checks. Validate before posting to Brightspace.

AI speeds up captioning and transcript work but does not finish accessibility. Provide captions or a transcript, apply the prompt, then complete the manual checklist and validation steps.

Fix or rebuild: choosing the most effective path

Accessibility remediation for media often starts with a decision. In many cases, rebuilding or replacing a recording is more efficient and produces better outcomes than repairing poor source material. The goal is not to preserve the original recording at all costs but to provide content that is perceivable, understandable, and usable for all learners.

  • Fix the media when the audio is clear and captions or a transcript are mostly accurate with correctable errors.
  • Rebuild or replace when the audio is difficult to understand, captions are severely inaccurate, or the recording is outdated and no longer instructionally necessary.

Both actions demonstrate a good-faith effort toward compliance and support an inclusive learning environment.

Reminder: replacement is an accessibility strategy

Video with captions

Typical uses include lecture recordings, demonstrations, guest talks, and narrated slide videos.

How to upload and use the prompts (Video)
  1. Use an institutionally approved AI tool. Do not upload sensitive student information.
  2. Prepare your source text. Captions or a transcript work best. If needed, export captions from your video platform or copy the transcript text.
  3. Provide the content in one of these ways:
    • Upload a caption or transcript file if file upload is available.
    • Paste caption or transcript text into the AI tool if file upload is not available.
  4. Paste one prompt from the prompt bank below.
  5. Retrieve the result and review it while playing the video to confirm timing cues, key terms, and speaker labels.
  6. Complete the manual remediation checklist and validation steps.
Prompt bank (Video)

Label: captions, transcript, summary

Prompt A. Caption cleanup and accuracy review

You may be receiving captions, a transcript, or pasted text generated from a video.

If readable caption or transcript text is present, edit it for accuracy, grammar, and clarity.
Preserve speaker changes and instructional emphasis.
Do not remove technical terms, names, or course specific vocabulary.

If no readable caption or transcript text is present, briefly state what content is needed to proceed.

Return revised caption or transcript text only.

Prompt B. Transcript formatting for accessibility

You may be receiving a transcript or unformatted text for a video.

If readable transcript text is present, format it for accessibility and readability.
Use clear paragraph breaks and consistent spacing.
Include speaker labels when they are present or instructionally important.
Do not summarize.

If no readable transcript text is present, briefly state what content is needed to proceed.

Return plain text only.

Prompt C. Student friendly summary for Brightspace

You may be receiving a transcript, captions, or pasted text.

If readable transcript or caption text is present, create a concise, student friendly outline suitable for a Brightspace page.
Preserve instructional sequence and key terms.
If timestamps are provided, include them for major topic shifts.
If timestamps are not provided, do not invent them.

If no readable transcript or captions are present, briefly state what content is needed to proceed.

Return plain text only.
Manual remediation checklist for Video
  • Ensure captions are present for required video.
  • Confirm caption accuracy for technical terms, names, and key concepts.
  • Confirm speaker identification when instructionally relevant.
  • Ensure non-speech audio that affects meaning is represented in text when appropriate.
  • Provide a transcript when captions are not available or not sufficient.
  • Ensure critical instructions are also available in text outside the video.
Validation for Video outputs
  • Play the video while reviewing captions to confirm alignment and accuracy.
  • If you post a transcript or summary in Brightspace, run the Ally Course Report for the page or document.
  • Spot check the posted Brightspace content with a screen reader.

Typical time: 10 to 30 minutes

Common pitfalls in Video
  • Auto captions that misrepresent technical terms and names.
  • Missing speaker labels in multi-speaker instruction.
  • Important information delivered only through visuals or audio with no text alternative.
  • Outdated recordings that are faster to replace than to repair.

Audio with transcripts

Typical uses include podcasts, audio lectures, interviews, and recorded feedback.

How to upload and use the prompts (Audio)
  1. Use an institutionally approved AI tool. Remove student identifiers and sensitive information.
  2. Start with text. If you only have audio, generate a transcript using your recording platform or transcription workflow first.
  3. Provide the transcript by uploading a file or pasting the transcript text.
  4. Paste one prompt from the prompt bank below.
  5. Review the output for accuracy, speaker labels, and key terminology.
  6. Complete the manual remediation checklist and validation steps.
Prompt bank (Audio)

Label: transcript cleanup and summary

Prompt A. Transcript cleanup and clarity review

You may be receiving a transcript or pasted text generated from audio.

If readable transcript text is present, edit it for accuracy, grammar, and clarity.
Preserve speaker changes and instructional emphasis.
Do not remove technical terms, names, or course specific vocabulary.

If no readable transcript text is present, briefly state what content is needed to proceed.

Return revised transcript text only.

Prompt B. Student friendly summary for Brightspace

You may be receiving a transcript or pasted text.

If readable transcript text is present, summarize it into a concise, student friendly outline suitable for a Brightspace page.
Preserve instructional sequence and key terms.

If no readable transcript text is present, briefly state what content is needed to proceed.

Return plain text only.
Manual remediation checklist for Audio
  • Provide a transcript for required audio.
  • Confirm accuracy of technical terms, names, and key concepts.
  • Include speaker identification when instructionally relevant.
  • Ensure the transcript is easy to locate in Brightspace and clearly labeled.
  • Avoid delivering critical instructions only through audio without a text version.
Validation for Audio outputs
  • Review the transcript while listening to the audio for accuracy.
  • If the transcript is posted as a Brightspace page or document, run the Ally Course Report.
  • Spot check with a screen reader to confirm readability and navigation.

Typical time: 10 to 25 minutes

Common pitfalls in Audio
  • Transcripts with missing sections due to poor audio quality.
  • Unclear attribution in multi-speaker audio.
  • Long transcripts posted without headings or spacing for readability.
  • Audio that is outdated and better replaced with a short written announcement.

Media summaries for Brightspace

Use summaries to help students preview, review, and locate key concepts. Summaries supplement captions and transcripts. They do not replace them.

How to upload and use the prompts (Summaries)
  1. Use an institutionally approved AI tool and remove student identifiers.
  2. Provide source text by uploading captions or a transcript, or pasting the text.
  3. Paste one prompt from the prompt bank below.
  4. Review the output for accuracy, terminology, and alignment to course learning objectives.
  5. Post the summary as a Brightspace page near the media item. Include a clear label such as "Video summary" or "Audio summary."
  6. Complete validation steps before publishing.
Prompt bank (Summaries)

Label: Brightspace ready text

Prompt A. Brightspace outline summary

You may be receiving a transcript, captions, or pasted text.

If readable transcript or caption text is present, create a structured outline for students.
Use short headings and bullet lists.
Preserve key terms and definitions.
If timestamps are provided, include them for major topic shifts.
If timestamps are not provided, do not invent them.

If no readable transcript or captions are present, briefly state what content is needed to proceed.

Return plain text only.

Prompt B. Study guide questions from a transcript

You may be receiving a transcript, captions, or pasted text.

If readable transcript or caption text is present, generate 6 to 10 study questions aligned to the content.
Include a mix of recall, application, and reflection questions.
Do not add information that is not present in the source text.

If no readable transcript or captions are present, briefly state what content is needed to proceed.

Return plain text only.
Manual remediation checklist for Summaries
  • Confirm summaries are accurate and aligned to the media content.
  • Ensure summaries do not introduce new facts that are not present in the recording.
  • Use headings and lists for readability.
  • Post summaries near the media item and label them clearly.
  • Maintain captions and transcripts as the primary accessibility support.
Validation for Summary outputs
  • Run the Ally Course Report for the Brightspace page.
  • Spot check with a screen reader for headings and list structure.
  • Confirm links, if included, have descriptive text.

Typical time: 5 to 15 minutes

Common pitfalls in Summaries
  • Summaries posted without clear labels or placed far from the media item.
  • Overly long paragraphs that should be structured with headings and lists.
  • Summaries treated as a substitute for captions or transcripts.

Third party media links

Typical uses include YouTube videos, news clips, and podcasts that you do not control. Provide clear labels, confirm caption or transcript status, and add a short student note that explains why the item is assigned.

How to upload and use the prompts (Third party)
  1. Use an institutionally approved AI tool. Do not upload copyrighted files you do not own.
  2. Start with text. If the platform provides captions or a transcript, copy that text. If not, consider replacing the item or providing an instructor-created summary that does not add new facts.
  3. Provide the URL and any available transcript by uploading a file or pasting the text.
  4. Paste one prompt from the prompt bank below.
  5. Review the output for accuracy, creator names, dates, and terminology.
  6. Complete the manual remediation checklist and validation steps.
Prompt bank (Third party)

Label: link note, citation, summary

Prompt A. Accessible use note and citation helper

You may be receiving a URL, captions, a transcript, or pasted text.

If a URL and readable text are present, draft a short Brightspace note that includes:
• Title, creator, publication date, and duration (if available)
• Caption or transcript status and where to find it on the platform
• One sentence on instructional purpose for this course
Then generate a complete citation in APA style using the available metadata.

If no readable text is present, briefly state what content is needed to proceed.

Return plain text only.

Prompt B. Student guide from a transcript

You may be receiving a URL plus captions, a transcript, or pasted text.

If readable transcript or caption text is present, create a concise outline for students with short headings and bullet lists.
Preserve key terms and definitions.
If timestamps are provided, include them for major topic shifts.
Do not invent timestamps.

If no readable transcript or captions are present, briefly state what content is needed to proceed.

Return plain text only.
Manual remediation checklist for Third party
  • Verify that captions or a transcript exist on the source platform. If not, consider replacing the item or posting a brief instructor summary near the link.
  • Label the link with the title, creator, and format, for example “Video” or “Podcast.”
  • Avoid autoplay and embedded players that add ads or trackers. Provide a standard hyperlink when needed.
  • Place the link near related course materials and include a one sentence purpose statement.
  • Ensure any summary posted in Brightspace does not add facts that are not in the source.
Validation for Third party outputs
  • Test the link in Brightspace to confirm it opens and is not region blocked.
  • Confirm captions or a transcript are available on the source page.
  • If you post a note or summary in Brightspace, run the Ally Course Report and spot check with a screen reader.

Typical time: 5 to 15 minutes

Common pitfalls in Third party
  • Dead or geo-restricted links with no alternative.
  • Auto captions that are inaccurate for technical terms.
  • Links that are not labeled with title, creator, and format.
  • Summaries that introduce new information not present in the source.

Audio descriptions for video

Use audio description when important information is conveyed visually and not available in dialogue or captions. Description can be built into the original narration, added as a separate track during pause points, or provided as a clearly labeled text description near the video.

How to upload and use the prompts (Audio description)
  1. Use an institutionally approved AI tool. Do not upload copyrighted files you do not own.
  2. Start with text. Obtain captions or a transcript first. Audio description is built from accurate source text.
  3. Provide the transcript by uploading a file or pasting the text, and include timestamps if available.
  4. Paste one prompt from the prompt bank below.
  5. Review the output for accuracy, terminology, and that visual information not spoken in the video is covered.
  6. Complete the manual remediation checklist and validation steps.
Prompt bank (Audio description)

Label: description script, student note

Prompt A. Create an audio description script from a transcript

You may be receiving a transcript, captions, or pasted text from a video.

If readable transcript or caption text is present, write an audio description script that narrates key visual information not spoken in the audio. Use neutral, concise language in present tense. Include:
• Actions, scene changes, on‑screen text, charts, and gestures that affect meaning
• Speaker identification when not clear from audio
• Placement notes like [insert during pause] only when necessary
If timestamps are provided, align description to major time points. Do not invent timestamps.

If there is no readable transcript or captions, briefly state what content is needed to proceed.

Return the audio description script as plain text.

Prompt B. Student note for Brightspace and choice of description method

You may be receiving a URL plus transcript or captions.

Draft a short student note for Brightspace that explains how audio description is provided for this video. Include:
• Method used (built‑in narration, separate description track, or text description on the page)
• Where to find the description (for example, the "AD" audio track in the player or the text below the video)
• One sentence on instructional purpose
Do not add new facts beyond the source.

Return plain text only.
Manual remediation checklist for Audio description
  • Confirm that meaningful visual information is described when not available in audio.
  • Prefer built‑in narration during production when possible. Use a separate track or text description when rebuilding is not practical.
  • Ensure technical terms, labels on charts, and on‑screen text are included when they affect learning.
  • Keep language concise and objective. Avoid interpreting or teaching new content in the description.
  • Clearly label the presence and location of the description in Brightspace or the media player.
Validation for Audio description outputs
  • Play the video without looking at the screen and confirm that all critical visuals are understood from audio plus description.
  • If using a separate description track, test track selection in the media player.
  • If using a text description on the page, run the Ally Course Report and spot check with a screen reader.

Typical time: 10 to 30 minutes

Common pitfalls in Audio description
  • Descriptions that repeat dialogue instead of covering visual information.
  • Missing description for on‑screen text, charts, or title slides.
  • Descriptions that are too long to fit in natural pauses.
  • Unlabeled or hard‑to‑find description tracks in the player.

Validation tools
Context Tool Action
Caption or transcript text Manual review with playback Confirm accuracy, speaker cues, and completeness
Brightspace pages for transcripts or summaries Ally Course Report Review flags and adjust headings, links, and structure
