PDF Accessibility: Remediation and Conversion Paths

Guidance for remediating PDFs for accessibility and converting them to more usable formats for Brightspace.

AI can assist with drafting structure, text alternatives, and conversions, but authors remain responsible for accuracy, instructional intent, and validation before publishing.

Copyright and accessibility

Remediating course materials for accessibility does not change their copyright status. Instructors may adapt, convert, or provide alternative formats of copyrighted materials for instructional use within a course when the purpose is to ensure equal access for enrolled students, consistent with accessibility obligations under Title II and related laws.

For questions about specific resources or edge cases, consult your library or institutional accessibility support team.

1. Text-based PDFs

AI-assisted conversion prompt 
You are assisting with accessibility remediation for a university course.
Convert this text-based PDF into semantic HTML for Brightspace.
Use one h1, ordered headings, descriptive link text, and accessible tables with captions and headers.
Provide alt text for images.
Return HTML only, followed by a short validation checklist.
  • Validate heading order and reading sequence
  • Confirm links describe destination and purpose
  • Run Ally after publishing in Brightspace

2. Image-based PDFs (scans)

AI-assisted OCR and rebuild prompt 
This PDF is a scanned document.
After OCR, rebuild it as accessible HTML for Brightspace.
Correct recognition errors.
Recreate tables with proper headers and captions.
Add alt text based on context.
Return HTML only with a validation checklist.
  • Manually verify OCR accuracy
  • Check math, symbols, and technical notation
  • Prefer rebuilding to tagging where quality is poor

3. Hybrid PDFs (text and figures)

AI-assisted hybrid conversion prompt 
Convert this hybrid PDF into accessible HTML for Brightspace.
Preserve text structure.
For each figure, include concise alt text and a short explanatory summary.
Add data tables where figures present quantitative information.
Return HTML only with a validation checklist.
  • Ensure figures are instructionally explained
  • Confirm tables include captions and headers
  • Spot-check with a screen reader

4. Interactive or form-fillable PDFs

AI-assisted form rebuild prompt 
Rebuild this PDF form as an accessible HTML form for Brightspace.
Use native form elements with labels.
Ensure logical tab order, visible focus, and clear error messages.
Avoid unnecessary ARIA.
Return HTML only with a validation checklist.
  • Keyboard navigation must be complete
  • Error messages must be programmatically associated
  • HTML forms are preferred over PDF forms

5. Already accessible PDFs

AI-assisted verification prompt 
Review this tagged PDF for accessibility.
Verify heading order, alt text accuracy, table headers, and link text.
If appropriate, convert to HTML for Brightspace.
Return HTML only with a validation checklist.
  • Do not assume automated checks are sufficient
  • Confirm tags align with visible structure
  • HTML improves usability for most learners

6. Student or instructor created PDFs

AI-assisted re-authoring prompt 
Rebuild this informal PDF as accessible HTML for Brightspace.
Add clear headings, summaries, and alt text.
Avoid layout tables.
Ensure logical reading order.
Return HTML only with a validation checklist.
  • Start from source files whenever possible
  • Avoid scanned submissions when alternatives exist
  • HTML is the preferred instructional format

Shared validation steps

  • Run Ally Course Report in Brightspace
  • Confirm keyboard navigation
  • Spot-check with a screen reader
  • Document remediation decisions and outcomes
