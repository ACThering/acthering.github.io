Accessibility Quick Tips Hub

Fast, practical guidance for professors building accessible Brightspace content.

Start with your file type, use the quick tips below, and expand sections only when you need more detail. The full guides can be added later as separate pages.

Start here in under 10 minutes

  1. Pick the content type you are working with.
  2. Apply the quick tips for that type.
  3. Run a quick validation check before you publish.

Goal: quick wins for accessibility

Common tasks professors ask about

  • Convert PDFs to accessible HTML.
  • Add alt text and long descriptions to images.
  • Fix table headers and structure.
  • Improve link text so it makes sense out of context.

Most fixes: 5–20 minutes

AI can accelerate drafting, but you are responsible for instructional accuracy and final validation. Use prompts to save time, then review in Brightspace.

Pick a content type

PDFs

Best for legacy documents. HTML is usually more usable in Brightspace.

  • Prefer converting to HTML for readability.
  • OCR and rebuild scanned PDFs.
  • Validate headings, tables, and links.
More PDF tips
  • Image-based scans should be rebuilt, not just tagged.
  • Hybrid PDFs need alt text and data tables for figures.
  • Run Ally Course Report after posting in Brightspace.

Page structure and headings

Use real headings and short sections so content is easy to scan.

  • One H1 title, then H2 sections.
  • Do not skip heading levels.
  • Use lists for steps and requirements.
More heading tips
  • Rebuild long pages into smaller chunks when topics shift.
  • Avoid bold text as a heading substitute.
  • Run the Brightspace accessibility checker when available.

Word, PowerPoint, Excel

Use built-in accessibility checkers and clean structure before upload.

  • Use real headings and list styles.
  • Fix reading order in PowerPoint.
  • Publish Excel data as HTML when possible.
More Office tips
  • Set document titles and language metadata.
  • Provide alt text or mark decorative images.
  • Use captions and headers for tables.

Video & audio

Captions and transcripts are the primary accessibility supports.

  • Review captions for accuracy.
  • Provide a transcript when needed.
  • Summaries help students preview content.
More media tips
  • Include speaker labels for multi-speaker audio.
  • Add audio description for key visuals.
  • Place summaries near the media item.

Images & alt text

Use purpose-based alt text and long descriptions for complex visuals.

  • Mark decorative images as decorative.
  • Write 1–2 sentence alt text for key visuals.
  • Use long descriptions for charts and infographics.
More image tips
  • Replace images of text with real text when possible.
  • Place long descriptions right below the image.
  • Verify alt text with a screen reader spot check.

Links

Link text should make sense without surrounding context.

  • Avoid “click here” and raw URLs.
  • Use the destination title or action.
  • Include file types for downloads.
More link tips
  • Use the same link text for the same resource.
  • Ensure keyboard focus is visible.
  • Run Ally for link-related feedback.

Tables & data

Tables are for comparisons, not layout. Keep them simple.

  • Use captions and headers.
  • Avoid merged cells.
  • Summarize key patterns in text.
More table tips
  • Convert layout tables into headings and lists.
  • Define units in header labels.
  • Spot check with a screen reader.

Shared validation steps

  • Run Ally Course Report after posting in Brightspace.
  • Use keyboard navigation to confirm logical focus order.
  • Spot check with a screen reader when possible.
  • Document what you fixed and what needs follow-up.
