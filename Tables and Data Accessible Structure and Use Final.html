Tables and Data: Accessible Structure and Use

Use AI to accelerate drafting and cleanup, then apply human review. Validate before posting to Brightspace.

AI can assist with drafting table structures and summaries, but authors remain responsible for clarity, accuracy, and instructional intent. Always validate headings, table headers, captions, and reading order in the final delivery format.

Decision guide: "Should this be a table?"

Tables support accessibility when they help learners compare values, identify relationships, or scan structured data. If you do not need comparisons or relationships, write the content as text instead. Rebuilding a table as clear text is often the most accessible option.

Choose the simplest format that supports learning and accessibility
If the content Use What to do
Requires comparing values across rows or columns Data table Use a real table with a clear header row (and header column if needed), plus a caption.
Lists items without comparison (for example, steps, supplies, policies) Text with headings and lists Write it as text using headings and bulleted or numbered lists.
Is only arranged in columns for layout or visual alignment Text layout (not a table) Rebuild as headings, paragraphs, and lists. Do not use tables for layout.
Is highly complex with merged cells or nested structures Rebuild Simplify into multiple smaller tables or convert key information into text and lists.

Quick check

Read each row out loud as if a screen reader is reading it: header, then each cell. If it does not make sense in that order, rebuild the table or switch to text.

Tables in Brightspace HTML

When you publish a table on a Brightspace page, use semantic HTML so assistive technologies can interpret relationships between headers and cells.

How to upload and use the prompts (Brightspace HTML table)
  1. Use an institutionally approved AI tool when working with university or student-related content.
  2. Remove student identifiers and sensitive information before uploading or pasting content.
  3. Provide your table content in one of these ways:
    • Upload a Word, Excel, or PDF file that contains the table, or
    • Paste the table content (including headers) into the prompt.
  4. Paste one prompt from the prompt bank below.
  5. Review the output for accuracy, especially header labels, units, and totals.
  6. Copy and paste the HTML into Brightspace and validate using Ally.
Prompt bank (Brightspace HTML table)

Label: semantic HTML tables

Prompt A. Convert a table into accessible Brightspace HTML

You may receive an uploaded file or pasted table content.

First, ask me:
1) What learners must do with this table (compare values, find patterns, answer a question)
2) The table title (caption) and any notes learners must know (units, abbreviations)

Then create an accessible HTML table for Brightspace:
- Include a <caption> that names the table and its purpose
- Use <thead> with <th scope="col"> for column headers
- Use <th scope="row"> if a row header is needed
- Do not use merged cells unless essential (and if used, explain what to verify)
- Keep header labels concise and meaningful

Return HTML code only, then a short validation checklist for headings, caption, and header associations.

Prompt B. Rebuild a table as accessible text when a table is not needed

You may receive an uploaded file or pasted content.

First, ask me what learners need from this content.
If learners do not need to compare values across rows or columns, rewrite the content as accessible text:
- Use headings and lists
- Preserve meaning and any required values
- Do not add new content

Return plain text suitable for a Brightspace HTML page, then a short validation checklist.

Prompt C. Add a short data summary for a table

You may receive an uploaded file or pasted table content.

Write a brief, student-facing summary that helps learners interpret the table:
- 2 to 4 sentences
- State the main pattern, range, or comparison the table supports
- Avoid conclusions not supported by the data

Return the summary only.
Manual checklist (Brightspace HTML table)
  • Include a caption that names the table and what it is for.
  • Confirm headers are real table headers, not bolded text.
  • Avoid merged cells. If merged cells are necessary, simplify the structure or split the table.
  • Keep header text short and specific (include units in header labels when helpful).
  • Use text near the table to define abbreviations or codes.
  • Run Ally Course Report after posting in Brightspace.
Validation (Brightspace HTML table)
  • Run Ally Course Report for the Brightspace page containing the table.
  • Spot check with a screen reader: confirm headers are announced with associated cells.
  • Keyboard check: confirm focus moves through the page logically.

Typical time: 10 to 30 minutes

Tables in Microsoft Word

Word tables are often used for schedules, rubrics, grading criteria, and policy summaries. Many accessibility issues come from missing header rows, merged cells, and layout tables.

How to upload and use the prompts (Word tables)
  1. Use an institutionally approved AI tool when working with university or student-related content.
  2. Remove student identifiers. Save a working copy of your document.
  3. Upload the .docx file (or paste the table content if upload is not available).
  4. Paste one prompt below. Each prompt assumes a file is attached or content is pasted.
  5. Review the output and apply the manual checklist before publishing.
Prompt bank (Word tables)

Label: Word table cleanup

Prompt A. Fix a Word table for accessibility

You may receive an uploaded Word document or pasted table content.

First, ask me what learners must do with the table.
Then improve the table for accessibility:
- Remove layout tables if present by converting to headings and lists
- Remove merged cells when possible
- Ensure a clear header row and consistent column labels
- Add a short table caption placed immediately above the table (plain text)

Return the revised table content and the caption text. If something cannot be fixed reliably, list what needs manual attention.

Prompt B. Convert a table to accessible Brightspace HTML (preferred for complex tables)

You may receive an uploaded Word document or pasted table content.

Convert the table into semantic HTML for Brightspace:
- Include a <caption>
- Use <thead> and <th scope="col"> for column headers
- Use <th scope="row"> if a row header is needed
- Avoid merged cells unless essential

Return HTML code only, then a short validation checklist.
Manual checklist (Word tables)
  • Use a real table only when learners must compare values or interpret relationships.
  • Keep tables simple: one header row, no merged cells when possible.
  • Write a short caption above the table (Word does not provide an HTML-style caption element).
  • Confirm any abbreviations are defined near the table.
  • Run Review, then Check Accessibility.
  • If the table is long, set the header row to repeat so screen readers and learners retain context.

Microsoft support: Repeat table header on subsequent pages

Simplified guidance

Check the option to repeat the header row so the header remains associated with the table content. This supports accessibility and reduces confusion when tables continue.

Validation (Word tables)
  • Run Microsoft Accessibility Checker in Word.
  • Confirm the table reads logically from headers to each row.
  • Consider converting complex tables to Brightspace HTML for better usability.

Typical time: 10 to 30 minutes

Data from Excel: publish for reading, not spreadsheets

Excel is powerful for analysis, but many spreadsheets are difficult to deliver accessibly as course content. For most courses, the highest payoff is to publish the data learners need in an accessible table (HTML or Word) plus a short summary, instead of posting a complex spreadsheet.

How to upload and use the prompts (Excel data)
  1. Use an institutionally approved AI tool when working with university or student-related content.
  2. Remove student identifiers. If the workbook includes grades or student work, do not upload it.
  3. Upload the .xlsx file or paste the relevant table range with headers.
  4. Paste one prompt below.
  5. Review for accuracy, especially labels, units, and totals.
  6. Publish the result in Brightspace and validate with Ally.
Prompt bank (Excel data)

Label: publish data accessibly

Prompt A. Convert Excel data to a Brightspace HTML table

You may receive an uploaded Excel file or pasted data.

First, ask me which sheet and which table range is the one to publish.
Then convert that data into a semantic HTML table for Brightspace:
- Include a clear <caption>
- Use <thead> and <th scope="col"> for column headers
- Avoid merged cells and blank spacer rows
- Keep header labels concise and meaningful

Return HTML code only, then a short validation checklist.

Prompt B. Draft a student-facing summary of the dataset

You may receive an uploaded dataset or pasted data.

Write a short summary for learners:
- 2 to 4 sentences
- State what the data represents, the time frame, and any important units
- State the main pattern learners should notice (if present)
- Avoid claims that are not supported by the data

Return the summary only.
Manual checklist (Excel data publishing)
  • Decide whether learners need a spreadsheet for interaction or whether they only need to read the data.
  • If the goal is reading, publish as HTML table plus a short summary.
  • Use clear headers and define units (percent, dollars, minutes) in headers or nearby text.
  • Avoid merged cells, hidden columns, and multiple unrelated tables on a single published table.
  • Validate the Brightspace page with Ally.
