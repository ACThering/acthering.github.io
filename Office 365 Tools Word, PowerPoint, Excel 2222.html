Fix or rebuild: choosing the most effective path

Accessibility remediation is not always about repair. In many cases, rebuilding a document from an accessible template is the most efficient and sustainable option. The goal is not to preserve the original layout at all costs but to create content that is structurally sound, perceivable, and maintainable.

  • Fix the file when it contains a few correctable issues, such as missing alt text, minor color contrast problems, or incomplete heading styles.
  • Rebuild when the document lacks semantic structure, includes images of text, or uses complex tables and layouts that cannot be corrected reliably.

Both actions demonstrate a good-faith effort toward compliance and support an inclusive learning environment.

Microsoft Word

Typical uses include syllabi, assignment sheets, lab instructions, and policy documents.

How to upload and use the prompts (Word)
  1. Open Copilot or a licensed ChatGPT account. Do not use the free ChatGPT version.
  2. Prepare your file. Remove student or personal data. Save a working copy.
  3. Attach the .docx using the paperclip or upload icon. Wait until the upload finishes.
  4. Paste one prompt from the prompt bank below.
  5. Retrieve the result. Download the returned .docx or copy any HTML the AI provides.
  6. Review the PowerPoint and add alt text to all informative images; mark decorative images as decorative.
  7. Complete the manual remediation checklist and validation steps.
Prompt bank (Word)

Label: conversion and HTML

Prompt A. Convert to accessible Word

You have my uploaded file. Convert it to an accessible .docx.
Use true heading styles in logical order, list semantics, descriptive link text,
table headers with scope where needed, image alt text, and language metadata.
Return the .docx only. If something cannot be remediated, list it on a final page.

Prompt B. Convert to Brightspace HTML

You have my uploaded document. Convert it to semantic HTML for Brightspace.
Use one h1, ordered headings, landmark regions, and descriptive links.
For tables, include <caption>, <thead>, and th scope attributes.
Return HTML code only, then a short validation checklist.
Manual remediation checklist for Word
  • Apply heading styles for all headings.
  • Use real lists, not typed hyphens or manual numbering.
  • Ensure table header rows and appropriate scope. Provide a caption and brief summary for data tables.
  • Replace images of text with real text or add precise alt text.
  • Check color contrast and avoid color-only meaning.
  • Set the document language.
  • Run Review → Check Accessibility and resolve remaining issues.
Validation for Word outputs
  • Microsoft Accessibility Checker for Word files.
  • If you posted HTML, run the Ally Course Report in Brightspace.
  • Spot check with a screen reader.

Typical time: 5 to 20 minutes

Common pitfalls in Word
  • Text appears visually like a heading but is not a heading style.
  • Decorative images have unnecessary alt text.
  • Complex tables that should be simplified or moved to HTML.
  • Do not rely on AI-generated alt text. Manually add alt text to informative images and mark decorative images appropriately.

Microsoft PowerPoint

Typical uses include lecture slides and narrated presentations.

How to upload and use the prompts (PowerPoint)
  1. Open Copilot or a licensed ChatGPT account with file upload enabled.
  2. Remove private data or embedded student examples from your slides.
  3. Upload the .pptx file and wait until the upload finishes.
  4. Paste one prompt from the prompt bank below.
  5. Download the returned .pptx and review reading order, alt text, and contrast.
  6. Complete the manual remediation checklist and validation steps.
Prompt bank (PowerPoint)

Label: conversion and outline

Prompt A. Convert to accessible PowerPoint

You have my uploaded deck. Convert it to an accessible .pptx.
Use one clear title per slide, logical reading order, sufficient contrast,
and alt text for images. Do not add speaker notes unless they exist.
Return the .pptx only. If something cannot be remediated, list it on a final slide.

Prompt B. PowerPoint to Brightspace HTML outline

You have my uploaded deck. Convert it into a concise HTML outline for Brightspace.
Use one h1 for the deck title, then h2 for slide titles, and bullet lists for key points.
Flag any images that need alt text. Return HTML code only, then a short validation checklist.
Manual remediation checklist for PowerPoint
  • Apply accessible slide layouts through the slide master.
  • Ensure each slide has a title placeholder.
  • Fix reading order for all content.
  • Provide alt text or mark decorative.
  • Check color contrast and avoid color-only meaning.
  • Avoid text embedded in images unless duplicated in real text.
Validation for PowerPoint outputs
  • Microsoft Accessibility Checker for PowerPoint.
  • If exported to HTML, run Ally in Brightspace and spot check with a screen reader.

Typical time: 10 to 30 minutes

Common pitfalls in PowerPoint
  • Stacked text boxes causing incorrect reading order.
  • Missing slide titles.
  • Small low-contrast text.
  • Do not rely on AI-generated alt text. Manually add alt text to informative images and mark decorative images appropriately.

Microsoft Excel

Decide early whether you are displaying data or delivering an interactive spreadsheet. If the goal is readability, convert key tables or charts to accessible HTML or Word.

How to upload and use the prompts (Excel)
  1. Open Copilot or a licensed ChatGPT account.
  2. Remove student data and unnecessary formulas or hidden sheets.
  3. Upload the .xlsx file and wait until the upload finishes.
  4. Paste the Excel prompt below for accessible HTML or Word conversion.
  5. Review the generated table or document for accuracy and logical structure.
  6. Run the Accessibility Checker or validate HTML output in Brightspace.
  7. Document your remediation actions.
Prompt bank (Excel)

Label: table conversions

Prompt A. Excel to accessible HTML table for Brightspace

You have my uploaded Excel file. Convert the key sheet into an accessible HTML table.
Include a <caption>, <thead>, and th scope="col". Use concise column headers.
Return HTML code and a short validation checklist for screen reader use.

Prompt B. Excel to accessible Word table

You have my uploaded Excel file. Convert the key sheet into an accessible Word table.
Use a single header row with clear column names, no merged cells, and a brief caption.
Return the .docx only and list any cells that need manual review.
Manual remediation checklist for Excel
  • Avoid merged cells and blank spacer rows.
  • Use a defined header row in each table.
  • Provide text labels for charts and data points.
  • Ensure adequate color contrast.
  • If interactivity is required, protect cells that are not intended for input and confirm tab order.
When to rebuild in Excel
  • Multiple tables on one sheet with merged cells.
  • Complex charts without labels.
  • Spreadsheets copied forward for years with inconsistent formatting.

Guidance: rebuild or convert to HTML for reading

Validation for Excel outputs
  • If staying in Excel, run the Accessibility Checker.
  • If you converted to Word or HTML, validate in that environment and spot check with a screen reader.

Accessible file names and document properties High payoff

File names and document properties are often the first information a screen reader announces. Clear names and correct metadata help students identify, organize, and return to course materials efficiently. This connects to Title II expectations without legal wording.

1. File names

Fastest improvement for access and organization.

  • Use descriptive, meaningful names students will recognize later.
  • Avoid defaults like Document1 or vague labels like Week3 or Scan.
  • Include the context students need such as course, term, topic, or version.

Examples

  • Bad: syllabus.pdf → Better: BIO101_Fall2025_Syllabus.pdf
  • Bad: Week4.pdf → Better: Week4_CellRespiration_Readings.pdf

Typical time: under 1 minute per file

2. Document title metadata (Word, PowerPoint, Excel)

The document title is not always the file name, and assistive technologies announce the title first.

  • Set a meaningful document title that matches the content students will see.
  • Do not leave defaults like Document1 or Microsoft Word – Untitled.

You do not need step-by-step clicks here. Just set a good title in the file properties before you upload.

3. Language setting

Set the document language so screen readers use correct pronunciation. This is especially important for bilingual or multilingual content.

  • Choose the primary language for the document.
  • If a section uses another language, mark that section’s language where your tool allows.

4. Versioning and updates

Reduce confusion for you and your students.

  • When you revise a document, update the file name or include a version tag such as v2 or a date.
  • Avoid uploading multiple different files with the exact same name in Brightspace.

Validation for file names and properties

  • Open the file and confirm the document title is correct.
  • Check the file name before uploading to Brightspace.
  • Run the built-in Accessibility Checker where available.

Typical time: 2 to 5 minutes per file

Common pitfalls

  • Files named Week1, Final, or Handout.
  • Document title left as a default value.
  • Mismatch between file name and document title.
  • Multiple uploads with identical names in Brightspace.
