Writing Clear Descriptive Links to Webpages and Resources

Write link text that communicates destination and purpose. Validate before posting to Brightspace.

Descriptive link text tells learners where a link goes and what to expect before they activate it. Screen reader users often navigate long pages by reviewing links out of context, so vague or generic link text limits comprehension and slows learning.

AI and author responsibility

AI can assist with drafting link text, but authors remain responsible for clarity, accuracy, and instructional intent.

Choosing descriptive link text decision guide

This applies when linking phrases like “click here,” “this page,” or entire sentences instead of naming the destination clearly.

Most learners benefit from clear link text, but this practice is especially critical for students using screen readers or keyboard navigation.

Quality checks you can do in under one minute
  • Out-of-context check: Read each link by itself and confirm it still makes sense.
  • Scan check: Skim only the linked phrases on the page and confirm you can understand the workflow without reading surrounding text.
  • Consistency check: Confirm repeated resources use the same link text across modules and weeks.
Manual checklist for descriptive links
  • Use meaningful link text that describes destination and purpose.
  • Keep link text concise while preserving key words.
  • Avoid using only “click here,” “more,” “this,” or the raw URL as the link text.
  • If the link initiates a task, include the action in the link text.
  • If the link points to a file, include the file type when helpful (PDF, Word, PowerPoint, Excel).
Prompt bank for drafting link text

Label: link text drafting support

Prompt A. Draft descriptive links from a list

You will receive a list of links I plan to include in a course page.
For each item, ask me for:
1) The intended learner task (what should the learner do)
2) The destination type (webpage, PDF, Word, Brightspace tool, video)
3) Any required wording (course terminology, assignment names)

Then draft descriptive link text for each item.
Requirements:
- Use the destination title or a meaningful summary
- Avoid generic phrases such as "click here" or "this page"
- Keep link text concise while preserving key words
- If it is a file, include the file type when helpful
Return the revised link text as a numbered list only.

Prompt B. Improve existing link text without changing meaning

You will receive existing link text and the destination description.
Rewrite each link to be more descriptive and task aligned without changing meaning.
Also do an out-of-context check: confirm each link makes sense by itself.
Return a two column list:
1) Current link text
2) Revised link text
Do not add new resources.

Reminder

Review all suggested changes before applying them, especially when link wording implies a specific action or task.

Validation in Brightspace
  • Run the Ally Course Report for the page and review any link related feedback.
  • Keyboard check: Use Tab to move through links and confirm focus is visible and the link order matches the page structure.
  • Screen reader spot check: Navigate by links and confirm the link list is understandable without surrounding context.

Typical time: 2 to 10 minutes per page

Common places faculty get stuck
  • Copying and pasting a full URL because it feels faster than naming the destination.
  • Linking an entire sentence instead of a concise destination phrase.
  • Using different names for the same resource across weeks, which confuses learners who skim.
  • Using link text that implies the wrong action (for example, “Download” when the link opens a webpage).
